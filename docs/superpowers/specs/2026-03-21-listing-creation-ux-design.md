# Listing Creation UX Improvement Design

## Overview

매물 등록 폼의 텍스트 입력을 최소화. DB에 383개 아이템 마스터 데이터(아이콘, 카테고리, 옵션, 강화 정보)가 있으나 거의 활용되지 않는 상태. 아이템 선택을 강화하고, 선택 후 자동 채움으로 사용자 입력량을 줄인다.

**접근**: 스마트 아이템 피커 — 기존 폼 구조 유지 + 아이템 선택 영역 대폭 강화 + 자동 채움.

## Current State

### 현재 등록 폼 필드 (11개)

1. 거래 유형 (판매/구매 토글)
2. 서버 (드롭다운, 27개)
3. 카테고리 (드롭다운, 상위 카테고리만)
4. 아이템명 (텍스트 검색 → 이름만 저장)
5. 강화 수치 (숫자 입력)
6. 제목 (텍스트, 필수)
7. 설명 (텍스트, 필수 10자+)
8. 가격 유형 (드롭다운)
9. 가격 (숫자)
10. 이미지 (파일 업로드)
11. 거래 방식 (드롭다운)

### 미활용 DB 데이터

`item_master` 테이블 (383개 아이템):
- `sub_category` — 세부 분류 (한손검, 양손검, 부츠 등)
- `option_text` — 아이템 기본 옵션 ("근거리 명중 +2" 등)
- `is_enchantable` — 강화 가능 여부
- `safe_enchant_level` — 안전 강화 수치
- `max_enchant_level` — 최대 강화 수치
- `icon_id` — 아이콘 이미지

### 현재 아이템 검색 API

`GET /items/search?q={query}&categoryId={optional}`

반환: `{ id, name, categoryId, iconUrl }` (4개 필드만)

## Design

### 1. 아이템 검색 API 확장

**변경**: `GET /items/search` 응답에 필드 추가

현재 반환:
```json
{ "id": "...", "name": "집행검", "categoryId": "weapon_sword", "iconUrl": "/static/icons/26.png" }
```

확장 후:
```json
{
  "id": "weapon_sword_집행검",
  "name": "집행검",
  "categoryId": "weapon_sword",
  "subCategory": "한손검",
  "iconUrl": "/static/icons/26.png",
  "optionText": "근거리 명중 +2",
  "isEnchantable": true,
  "safeEnchantLevel": 6,
  "maxEnchantLevel": 10
}
```

**백엔드 변경**: `handlers_master.go`의 searchItems 핸들러에서 `item_master` 테이블의 추가 컬럼 SELECT. `is_enchantable` INTEGER(0/1) → JSON boolean 변환 필요 (`item.IsEnchantable != 0`).

**카테고리 필터 미입력 시**: 검색어 없이 `categoryId`만 전달하면 해당 카테고리 아이템 목록 반환 (현재는 `q` 필수 1자 이상 → 빈 `q`도 허용하도록 변경, `categoryId`가 있으면).

**카테고리 경로 표시**: UI에서 "무기 > 한손검" 경로를 표시할 때, `subCategory` 값은 `categories` 테이블의 `name`과 동일하므로 프론트에서 이미 로드된 categories 데이터로 부모 카테고리명을 resolve한다. API에 별도 필드 추가 불필요.

### 2. 프론트엔드 타입 확장

`ItemSearchResult` 타입 추가 또는 기존 타입 확장:

```typescript
interface ItemSearchResult {
  id: string;
  name: string;
  categoryId: string;
  subCategory?: string;
  iconUrl?: string;
  optionText?: string;
  isEnchantable: boolean;
  safeEnchantLevel?: number;
  maxEnchantLevel?: number;
}
```

### 3. 아이템 피커 컴포넌트 재설계

기존 `item-autocomplete.tsx`를 확장하여 3가지 상태를 가진 컴포넌트로 변경.

#### 상태 A: 미선택 (검색 모드)

```
┌──────────────────────────────────────┐
│ 카테고리: [무기][방어구][악세][소모]...│
│ 서브:     [한손검][양손검][단검]...    │
│ ┌────────────────────────────────┐   │
│ │ 🔍 아이템 검색...              │   │
│ └────────────────────────────────┘   │
│                                      │
│ ┌────┐ ┌────┐ ┌────┐ ┌────┐        │
│ │icon│ │icon│ │icon│ │icon│ ...     │
│ │집행│ │진명│ │커스│ │데스│         │
│ │검  │ │황의│ │타드│ │나이│         │
│ └────┘ └────┘ └────┘ └────┘        │
└──────────────────────────────────────┘
```

- **상단**: 카테고리 탭 (수평 스크롤, 상위 카테고리 전체 — 현재 10개)
- **하위**: 서브카테고리 칩 (탭 선택 시 해당 서브카테고리 표시)
- **검색**: 텍스트 입력. 카테고리 필터 + 검색어 동시 적용
- **결과**: 그리드 카드 형태 (아이콘 중심). 카테고리 선택만으로도 아이템 목록 표시 (검색어 없이)
- **카드**: 아이콘(40x40) + 이름 + 옵션 한 줄

#### 상태 B: 아이템 선택됨 (카드 모드)

```
┌──────────────────────────────────────┐
│ [아이콘]  집행검                [변경]│
│           무기 > 한손검              │
│           근거리 명중 +2             │
│                                      │
│  강화: ━━━━━●━━━ +7                  │
│        0   ↑6(안전)    10            │
└──────────────────────────────────────┘
```

- 선택한 아이템의 상세 카드 표시
- 아이콘 크게 (48x48)
- 카테고리 경로 (무기 > 한손검)
- 옵션 텍스트
- 강화 슬라이더 (is_enchantable일 때만):
  - range: 0 ~ maxEnchantLevel
  - 안전 강화 위치에 시각적 마커
  - 현재 값 표시: `+7`
- "변경" 버튼 → 상태 A로 복귀

#### 상태 C: 강화 불가 아이템

```
┌──────────────────────────────────────┐
│ [아이콘]  빨간 물약             [변경]│
│           소모품                      │
│           회복량 6 ~ 27              │
└──────────────────────────────────────┘
```

강화 슬라이더 없음. 카드만 표시.

### 4. 자동 채움 로직

아이템 선택 시:

| 폼 필드 | 자동 채움 값 | 조건 |
|---------|------------|------|
| `categoryId` | `item.categoryId` | 항상 |
| `enhancementLevel` | `0` (기본값) | `isEnchantable`일 때만 표시 |
| `title` | `"{이름} +{강화} {판매/구매}합니다"` | 아이템 선택 시 초기 생성 |
| `description` | `"[아이템 옵션]\n{optionText}\n\n"` | `optionText` 있을 때 |

### 5. 제목 자동 생성 규칙

패턴: `"{아이템명}{강화표시} {거래유형}합니다"`

- 강화 0 또는 비강화 아이템: `"집행검 판매합니다"`
- 강화 있음: `"집행검 +7 판매합니다"`
- 거래유형 변경 시: `"집행검 +7 구매합니다"`

자동 생성 추적:
- `titleAutoGenerated` 상태값으로 추적
- 사용자가 제목을 수동 편집하면 `false`로 전환 → 이후 아이템/강화 변경 시 제목 유지
- 아이템을 다시 변경하면 `true`로 리셋 → 새 제목 자동 생성

### 6. 설명 자동 프리필

아이템 선택 시 `optionText`가 있으면:

```
[아이템 옵션]
근거리 명중 +2

```

- 사용자가 아래에 추가 설명 작성
- 옵션 텍스트 자체가 10자 미만이면 placeholder로 "(추가 설명을 입력하세요)" 안내
- 사용자가 설명을 완전히 지우고 새로 작성해도 OK

### 7. 폼 필드 순서 변경

**개선 후 순서**:

1. **거래 유형** — 판매/구매 토글 (기존과 동일)
2. **아이템 선택** — 스마트 피커 (카테고리 탭 + 검색 + 그리드 + 선택 카드 + 강화 슬라이더)
3. **서버** — 드롭다운 (기존과 동일)
4. **가격** — 타입 드롭다운 + 금액 입력 (기존과 동일)
5. **제목** — 자동 생성됨, 편집 가능
6. **설명** — 옵션 프리필, 편집 가능
7. **이미지** — 파일 업로드 (기존과 동일)
8. **거래 방식** — 드롭다운 (기존과 동일)

**제거**: 카테고리 독립 드롭다운 (아이템 선택 시 자동 결정)

### 8. 제출 데이터

백엔드 `handleCreateListing` 요청 구조는 변경 없음. 프론트에서 자동 채운 값을 기존 필드에 매핑:

- `categoryId` ← `selectedItem.categoryId` (자식 카테고리 ID, 예: `weapon_sword`)
  - 참고: 기존 등록은 부모 카테고리(예: `weapon`)를 전송했으나, 자식 카테고리도 유효함
  - 매물 필터 SQL이 `WHERE category_id = $N OR category_id IN (SELECT id FROM categories WHERE parent_id = $N)` 패턴으로 양쪽 모두 처리
  - 자식 카테고리 사용이 더 정확한 분류이므로 의도적으로 변경
- `itemName` ← `selectedItem.name` (기존처럼 이름 문자열)
- `enhancementLevel` ← 슬라이더 값
- `title` ← 자동 생성 또는 사용자 편집
- `description` ← 프리필 + 사용자 편집
- 나머지 필드 그대로

백엔드 변경은 검색 API 확장만 필요. 등록 API는 변경 없음.

| `backend/cmd/server/handlers_master.go` | searchItems 응답에 6개 필드 추가, 빈 검색어 허용 (categoryId 있을 때) |
| `backend/internal/repository/postgres_master.go` | searchItems SQL에 컬럼 추가 |
| `backend/internal/repository/interfaces.go` | Go `ItemSearchResult` 구조체에 필드 추가 |
| `web/lib/types.ts` | `ItemSearchResult` 타입 추가/확장 |

| `web/app/create/page.tsx` | 자동 채움, 제목 생성, 필드 순서 변경, 카테고리 독립 필드 제거 |

## Out of Scope

- 아이템 즐겨찾기 / 최근 사용 아이템
- 가격 시세 참조 (별도 피처)
- 아이템 이미지 크롤링 (기존 icon_id 활용)
- 매물 수정 페이지 (등록 개선 후 별도 적용)
- 서버 선택 UX 개선 (현재 드롭다운으로 충분)
