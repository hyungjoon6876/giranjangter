# 기란JT 웹 UX 분석 리포트

> 분석일: 2026-03-17 | 대상: https://giranjt.com | 분석 도구: Playwright MCP
> 분석 범위: 비로그인 사용자 관점 (Desktop 1280px + Mobile 390x844)

---

## 종합 평가

기란JT 웹은 리니지 클래식 거래 플랫폼으로서 핵심 기능(매물 목록, 상세, 채팅, 매물 등록)을 잘 갖추고 있다. 다크 테마 + 골드 강조색 조합이 게임 커뮤니티 타겟에 적합하며, 접근성(skip link, aria, focus trap)도 기본적으로 잘 구현되어 있다.

**강점:**
- 게임 커뮤니티에 맞는 시각 디자인 (다크 테마 + 골드/크림 강조)
- 모바일/데스크톱 반응형 레이아웃
- 접근성 기본 구현 (skip link, aria-label, focus trap modal)
- 빈 상태/에러 상태 일관된 처리
- 검색 결과 빈 상태에서 검색어 표시

**약점:**
- 비로그인 UX 흐름이 약함 (auth wall이 많고 피드백 부족)
- 서버 필터에 서버가 "전체"밖에 없음
- Hero 섹션이 매물 목록을 밀어냄
- 일부 핵심 백엔드 기능이 웹에 미노출

## 1. 페이지별 UX 분석

### 1.1 홈페이지 (마켓)

**데스크톱 (1280px)**
- Hero 섹션이 전체 above-the-fold를 차지하여 실제 매물 목록이 스크롤 없이 보이지 않음
- 3개 Feature 카드(무료 거래, 신뢰 시스템, 실시간 채팅)는 정보량이 적어 공간 대비 가치가 낮음
- "매물 둘러보기" 버튼이 `#listings` 앵커로 스크롤하지만, 첫 방문자에겐 비직관적
- 서버 필터에 "전체" 버튼만 존재 → 서버 데이터가 없거나 API 이슈

**모바일 (390px)**
- Bottom nav 잘 구현 (마켓/채팅/등록/프로필)
- FAB(+) 버튼이 있으나 용도가 불명확 (매물 등록인지?)
- Hero 섹션이 더 많은 스크롤을 요구

**이슈:**

| # | 심각도 | 이슈 | 개선안 |
|---|--------|------|--------|
| H1 | 높음 | Hero가 매물 목록을 가림 — 재방문 사용자에겐 불필요 | 로그인 사용자에겐 Hero 축소/제거, 매물 목록 바로 노출 |
| H2 | 높음 | 서버 필터에 서버 없음 | 서버 목록 API 연동 확인, 서버 데이터 표시 |
| H3 | 중간 | Feature 카드 공간 과다 사용 | 한 줄 배너 or 접기 가능한 UI로 압축 |
| H4 | 중간 | 매물 없을 때 빈 상태가 밋밋 | 인기 카테고리 추천, 최근 거래 완료 등 대체 콘텐츠 |
| H5 | 낮음 | 검색바가 Header + 매물 목록에 중복 존재 | 하나로 통합 or 역할 분리 명확화 |

### 1.2 로그인 페이지

**데스크톱/모바일 공통**
- 중앙 정렬 로고 + Google 로그인 버튼 + "둘러보기" 링크
- 디자인이 깔끔하고 단순

**이슈:**

| # | 심각도 | 이슈 | 개선안 |
|---|--------|------|--------|
| L1 | 높음 | Google OAuth만 지원 — 구글 계정 없는 사용자 차단 | 카카오/네이버 OAuth 추가 (한국 게임 커뮤니티 특성) |
| L2 | 중간 | "둘러보기" 버튼이 뒤로가기와 동일한 동작인지 불명확 | "로그인 없이 매물 둘러보기"로 명확한 문구 |
| L3 | 중간 | 왜 로그인해야 하는지 설명 없음 | "거래를 시작하려면 로그인이 필요합니다" 등 컨텍스트 제공 |
| L4 | 낮음 | 로그인 페이지에서 화면 대부분이 빈 공간 | 플랫폼 소개/혜택을 사이드에 배치 (데스크톱) |

### 1.3 채팅/매물등록 (인증 필요)

**현재 동작:**
- `/create` → 로그인 페이지로 리다이렉트 (아무 피드백 없이)
- `/chats` → 로그인 페이지로 리다이렉트 (아무 피드백 없이)

**이슈:**

| # | 심각도 | 이슈 | 개선안 |
|---|--------|------|--------|
| A1 | 높음 | 보호된 페이지 접근 시 무언의 리다이렉트 | 토스트 메시지 "채팅은 로그인이 필요합니다" + 로그인 후 원래 페이지로 리다이렉트 |
| A2 | 높음 | 로그인 후 돌아갈 URL이 저장되지 않는 것으로 보임 | `?redirect=/create` 파라미터로 로그인 후 복귀 |
| A3 | 중간 | Nav에서 "매물 등록"이 보이지만 클릭하면 로그인으로 이동 — 혼란 유발 | 비로그인 시 버튼에 잠금 아이콘 표시 or 클릭 시 모달로 안내 |

### 1.4 알림 페이지

**데스크톱/모바일 공통**
- 비로그인에서도 접근 가능 (로그인 리다이렉트 없음)
- "알림이 없습니다" 빈 상태 표시

**이슈:**

| # | 심각도 | 이슈 | 개선안 |
|---|--------|------|--------|
| N1 | 중간 | 비로그인인데 알림 페이지 접근 가능 — 항상 빈 상태 | 비로그인 시 "로그인하면 알림을 받을 수 있습니다" 안내 |
| N2 | 낮음 | 빈 상태 아이콘이 돋보기(🔍) — 알림과 무관 | 종 아이콘(🔔)으로 변경 |

### 1.5 프로필 페이지

**데스크톱/모바일 공통**
- 비로그인 시 "로그인이 필요합니다" + "로그인하기" 링크
- 단순하지만 기능적

**이슈:**

| # | 심각도 | 이슈 | 개선안 |
|---|--------|------|--------|
| P1 | 중간 | 프로필 수정 기능 없음 (백엔드 API 존재하나 웹 미구현) | 프로필 편집 UI 추가 (닉네임, 소개) |
| P2 | 중간 | 사용자 리뷰 조회 UI 없음 (백엔드 API 존재) | 프로필에 받은 리뷰 탭 추가 |
| P3 | 낮음 | "로그인이 필요합니다" 페이지가 시각적으로 빈약 | 로그인 혜택 설명 + 시각적 개선 |

---

### 1.6 매물 상세 페이지

**코드 분석 기반 (현재 매물 0개로 라이브 확인 불가)**
- 타입 뱃지, 상태 뱃지, 가격, 아이템 정보, 거래 정보 카드, 작성자 카드 구성
- 액션 바: 찜하기, 채팅하기, 신고
- 이미지 갤러리 없음 (아이템 아이콘만 표시)

**이슈:**

| # | 심각도 | 이슈 | 개선안 |
|---|--------|------|--------|
| D1 | 높음 | 이미지 업로드/갤러리 없음 (백엔드 API 존재하나 웹 미구현) | 매물 이미지 업로드 + 상세 갤러리 뷰 |
| D2 | 중간 | 찜 버튼 텍스트가 "관심"/"관심"으로 동일 (찜 여부 구분 불가) | "♡ 관심" / "♥ 관심" or 아이콘 색상 변경 |
| D3 | 중간 | 공유 기능 없음 | URL 복사/SNS 공유 버튼 추가 |
| D4 | 낮음 | 조회수/찜수 카운터가 상세 페이지에 표시되지 않음 | 조회수, 찜 수, 채팅 수 표시 |

### 1.7 검색

**현재 동작:**
- Header 검색바 + 매물 목록 내 검색바 (2개 존재)
- 검색 결과 없을 때 잘 처리 ("집행검"에 대한 매물을 찾을 수 없습니다)
- 검색어 클리어 버튼(×) 제공

**이슈:**

| # | 심각도 | 이슈 | 개선안 |
|---|--------|------|--------|
| S1 | 중간 | 검색 자동완성/추천 없음 | 아이템 마스터 데이터 기반 자동완성 (`/items/search` API 존재) |
| S2 | 중간 | 카테고리/정렬 필터가 UI에 없음 | 무기/방어구/기타 카테고리 + 가격순/최신순 정렬 추가 |
| S3 | 낮음 | Header 검색과 매물 목록 검색의 관계 불명확 | 통합 or 각각의 역할 분리 |

## 2. 미구현 기능 분석 (백엔드 API 존재, 웹 미노출)

백엔드에 API가 구현되어 있으나 웹 프론트엔드에서 접근할 수 없는 기능들:

| 기능 | 백엔드 API | 웹 구현 상태 | 우선순위 |
|------|-----------|-------------|----------|
| 프로필 수정 | `PATCH /me/profile` | 미구현 | P1 |
| 이미지 업로드 | `POST /uploads/images` | 미구현 | P0 |
| 사용자 차단 | `POST/DELETE /users/:id/block` | 미구현 | P2 |
| 사용자 리뷰 조회 | `GET /users/:id/reviews` | 미구현 | P1 |
| 내 신고 내역 | `GET /me/reports` | 미구현 | P3 |
| 아이템 검색 (마스터) | `GET /items/search` | 미구현 | P1 |
| 매물 상태 변경 | `POST /listings/:id/status` | 미구현 | P1 |
| 매물 수정 | `PATCH /listings/:id` | 미구현 | P0 |
| 예약 확인/취소 | `POST /reservations/:id/confirm\|cancel` | 미구현 | P1 |
| 거래 완료 확인 | `POST /trade-completions/:id/confirm` | API만 연동 | P1 |

## 4. 접근성 (a11y) 평가

**잘 된 부분:**
- "본문으로 건너뛰기" skip link 구현
- `aria-label` 적용 (메인 메뉴, 하단 메뉴, 매물 액션 toolbar)
- Modal focus trap + Escape 닫기 + inert siblings
- `aria-pressed` on 찜 버튼
- `role="status"` on 매물 목록 로딩

**개선 필요:**

| # | 이슈 | 개선안 |
|---|------|--------|
| A11Y-1 | 알림 아이콘에 `img` alt 속성 없음 | `alt="알림"` 추가 |
| A11Y-2 | Feature 카드에 시멘틱 마크업 없음 (generic div) | `<article>` 또는 headings 추가 |
| A11Y-3 | 서버 필터 버튼 그룹에 개별 서버 aria 설명 없음 | 현재는 서버가 하나뿐이라 낮은 우선순위 |
| A11Y-4 | 검색 결과 변경 시 screen reader 안내 없음 | `aria-live="polite"` 결과 카운트 영역 |

## 5. 우선순위 액션 플랜

### Phase 1 — 핵심 사용성 개선 (즉시)
1. Auth wall 개선: 로그인 후 복귀 URL + 토스트 메시지
2. Hero 섹션 축소/조건부 표시 (재방문자에게는 매물 바로 노출)
3. 찜 버튼 아이콘/텍스트 구분 수정
4. 서버 필터 데이터 연동 확인

### Phase 2 — 기능 완성 (단기)
5. 매물 이미지 업로드 + 갤러리
6. 매물 수정 기능
7. 카테고리 필터 + 정렬 UI
8. 아이템 자동완성 검색

### Phase 3 — 신뢰 시스템 강화 (중기)
9. 사용자 리뷰 표시
10. 프로필 수정
11. 매물 상태 관리 UI
12. 카카오/네이버 OAuth

### Phase 4 — 완성도 (장기)
13. 매물 공유 기능
14. 사용자 차단
15. 푸터 + 법적 페이지
16. PWA 지원
