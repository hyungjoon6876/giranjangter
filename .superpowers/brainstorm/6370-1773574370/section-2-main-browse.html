섹션 2: 메인 탐색 경험

매물을 찾고, 필터링하고, 탐색하는 과정의 접근성과 UX를 개선합니다.

2-A. 검색 접근성 강화

현재
아이템 검색...

✗ label 없음 (placeholder만 의존)
✗ aria-label 없음
✗ 검색 결과 카운트 안내 없음
✗ PC에서 Ctrl+K 단축키 없음

개선
🔍 아이템 검색... ⌘K
23개 매물

✓ aria-label="매물 검색"
✓ role="searchbox"
✓ 결과 카운트 aria-live="polite"
✓ Ctrl+K / ⌘K 포커스 단축키

2-B. 필터 칩 키보드 접근성

현재 문제: 서버 필터 버튼에 aria-current 없음. 선택 상태가 시각적으로만 표현됨.

개선:

  • 버튼 그룹에 role="group" + aria-label="서버 필터"
  • 선택된 필터에 aria-current="true"
  • focus-visible:ring-2 focus-visible:ring-gold 포커스 링 추가
  • 필터 변경 시 결과 카운트를 aria-live="polite" 영역으로 안내

2-C. 결과 카운트 + 정렬 컨트롤

필터 아래 정보 바 추가
23개 매물

✓ 네이티브 <select> → 키보드/스크린 리더 완벽 지원
✓ label 연결: aria-label="정렬 방식"

2-D. 매물 카드 접근성

현재 문제: 카드 링크에 aria-label 없음, 타이틀 truncate 시 전체 텍스트 접근 불가, focus-visible 스타일 없음

개선:

  • 카드 Link에 aria-label="{아이템명} - {가격} - {서버명}" 조합 라벨
  • truncated 텍스트에 title 속성으로 전체 텍스트 접근
  • focus-visible:ring-2 focus-visible:ring-gold focus-visible:ring-offset-2
  • 카드 호버/포커스 시 동일한 시각적 피드백 (outline glow)
  • 아이콘 이미지에 의미 있는 alt="{아이템명} 아이콘"

2-E. 컨텍스트 빈 상태

현재: 항상 동일한 "매물이 없습니다" 메시지

개선:

  • 검색 결과 없음: "'{검색어}'에 대한 매물이 없습니다"
  • 필터 결과 없음: "선택한 서버에 매물이 없습니다. 다른 서버를 선택해보세요."
  • 진짜 빈 상태: "아직 매물이 없습니다. 첫 매물을 등록해보세요!"
  • EmptyState 컴포넌트의 타이틀을 <h2>로 변경 (시맨틱 계층)
  • 아이콘 emoji에 role="img" + aria-label
