5-C. 폼 검증 접근성

현재: 매물 등록 폼에서 에러 메시지가 스크린 리더에 전달되지 않음

개선:

  • 각 input에 aria-describedby로 에러 메시지 연결
  • 에러 메시지에 role="alert" → 즉시 안내
  • 에러 발생 시 첫 번째 에러 필드로 포커스 이동
  • 필수 필드에 aria-required="true"
  • 에러 상태 input에 aria-invalid="true"

5-D. prefers-reduced-motion 대응

현재: 스피너 애니메이션, 호버 트랜지션이 모션 감소 설정을 무시함

개선:

  • globals.css에 @media (prefers-reduced-motion: reduce) 추가
  • 애니메이션 → 즉시 전환 또는 비활성화
  • 스피너 → 정적 "로딩 중..." 텍스트로 대체
  • 카드 hover 효과 (translate, glow) → 배경색 변경만

5-E. globals.css 접근성 유틸리티

현재: sr-only, focus-visible 등 접근성 유틸리티 클래스 없음

추가할 유틸리티:

  • .sr-only — 시각적으로 숨기되 스크린 리더에는 노출
  • .not-sr-only — sr-only 해제 (skip link 포커스 시)
  • .skeleton — 스켈레톤 로딩 펄스 애니메이션
  • .focus-gold — 통일된 focus-visible 스타일
