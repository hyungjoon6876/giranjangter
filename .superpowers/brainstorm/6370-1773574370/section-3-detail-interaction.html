섹션 3: 상세 페이지 + 상호작용

매물 상세, 채팅, 모달 등 사용자가 깊이 들어가는 단계의 접근성과 UX를 개선합니다.

3-A. 모달 접근성 (가장 심각한 이슈)

현재: 접근성 위반

✗ role="dialog" 없음
✗ aria-modal="true" 없음
✗ aria-labelledby 없음
✗ 포커스 트랩 없음 (Tab이 모달 밖으로 탈출)
✗ Escape 키로 닫기 불가
✗ 닫기 버튼 "×"에 aria-label 없음
✗ 닫힌 후 포커스가 원래 위치로 안 돌아감

개선: WCAG 준수 모달

✓ role="dialog" + aria-modal="true"
✓ aria-labelledby → 모달 타이틀 연결
✓ 포커스 트랩 (Tab/Shift+Tab 순환)
✓ Escape 키 닫기
✓ aria-label="닫기" 버튼
✓ 닫힌 후 트리거 요소로 포커스 복귀
✓ 배경에 aria-hidden="true" inert 적용

→ 예약 모달, 리뷰 모달, 신고 모달 모두 동일한 Modal 컴포넌트를 사용하므로 한 번 수정으로 전체 적용

3-B. 매물 상세 페이지 접근성

개선:

  • 아이템 정보를 <dl> (정의 목록) 구조로 변경 — <dt>서버, <dd>바츠
  • 하단 액션 바에 role="toolbar" + aria-label="매물 액션"
  • 찜 버튼 상태 구분: aria-pressed="true/false" + aria-label="찜하기" / "찜 취소"
  • 채팅 시작 버튼의 alert() → Toast로 교체
  • 로딩 실패 시 에러 UI + "다시 시도" 버튼 추가

3-C. 채팅 실시간 UX + 접근성

채팅 영역 접근성 개선

메시지 영역

✓ role="log" — 메시지 목록이 시간순 로그임을 명시
✓ aria-live="polite" — 새 메시지 도착 시 스크린 리더 안내
✓ 시스템 메시지에 role="status"
✓ 각 메시지에 시간 정보 (sr-only로 숨김 가능)

입력 영역

✓ aria-label="메시지 입력"
✓ 전송 버튼 aria-label="메시지 전송"
✓ PC: Shift+Enter = 줄바꿈, Enter = 전송
✓ 전송 실패 시 Toast + 재전송 버튼

채팅 목록

✓ 활성 채팅에 aria-current="true"
✓ 안 읽은 메시지 뱃지 aria-label="N개 안 읽은 메시지"
✓ 채팅방 선택 시 메시지 입력으로 auto-focus

3-D. SSE 연결 상태 표시

현재: SSE 연결이 끊어져도 사용자에게 아무 피드백 없음. 무한 재시도.

개선:

  • useSSE 훅에 isConnected / isReconnecting 상태 노출
  • 연결 끊김 시 채팅 상단에 "연결이 끊어졌습니다. 재연결 중..." 배너
  • 재연결 최대 횟수 제한 (10회) + 지수 백오프 (1s → 2s → 4s...)
  • 배너에 role="alert"로 스크린 리더 즉시 안내
