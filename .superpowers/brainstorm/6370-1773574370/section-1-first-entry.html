섹션 1: 첫 진입 경험

사용자가 기란장터에 처음 도착했을 때 "뚝 떨어지는 느낌"을 해결합니다.

1-A. 스켈레톤 로딩 (스피너 → 레이아웃 예약)

현재: 스피너만

빈 화면 + 스피너

레이아웃이 예약되지 않아
콘텐츠 등장 시 화면이 점프

개선: 스켈레톤 카드

role="status" aria-label="매물 목록을 불러오는 중"

1-B. 자연스러운 로그인 진입점

현재: 헤더에 프로필 아이콘만 있고, 비로그인 사용자에게 로그인 유도가 약함

개선 안:

  • 헤더 우측에 "로그인 / 시작하기" 텍스트 버튼 표시 (비로그인 시)
  • 히어로 섹션의 CTA "시작하기" 버튼을 더 눈에 띄게 개선
  • 매물 카드에서 "찜하기" 클릭 시 → 로그인 모달로 자연스럽게 유도
  • 로그인 페이지의 alert() → 인라인 에러 메시지로 교체

1-C. Loading 컴포넌트 접근성

현재 문제: <Loading />에 role="status"도, aria-label도 없어서 스크린 리더에 아무 정보 없음

개선:

  • role="status" + aria-label="로딩 중" 추가
  • aria-busy="true"를 부모 컨테이너에 적용
  • prefers-reduced-motion 미디어 쿼리 대응 (애니메이션 비활성화)
  • .sr-only 유틸리티 클래스 globals.css에 추가

1-D. 에러 상태 → alert() 제거, 토스트 도입

현재 문제: 로그인 실패, 채팅 시작 실패 등에서 alert() 사용 → 브라우저 차단 + 접근성 문제

개선:

  • Toast 컴포넌트 신규 생성 (components/ui/toast.tsx)
  • role="alert" + aria-live="assertive"로 스크린 리더 즉시 안내
  • 자동 사라짐 (5초) + 수동 닫기 버튼
  • 성공(green)/에러(red)/정보(gold) 3가지 변형
  • 모든 alert() 호출을 toast로 교체
