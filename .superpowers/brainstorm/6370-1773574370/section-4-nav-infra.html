섹션 4: 네비게이션 인프라

헤더, 하단 탭바, 사이드바 등 전체 앱을 감싸는 네비게이션의 접근성을 개선합니다.

4-A. Skip-to-Content 링크

키보드 첫 Tab 시 나타나는 링크
본문으로 건너뛰기

→ 평소에는 화면 밖에 숨겨져 있음 (sr-only + focus:not-sr-only)
→ Tab 키를 누르면 화면 상단에 나타남
→ 클릭/Enter 시 main 콘텐츠로 포커스 이동
→ WCAG 2.4.1 필수 요구사항

4-B. 헤더 접근성

개선:

  • <header role="banner"> 랜드마크 추가
  • 데스크탑 네비 링크: <nav aria-label="메인 메뉴">
  • 활성 페이지 링크: aria-current="page"
  • 알림 아이콘: aria-label="알림" + 뱃지 있으면 aria-label="알림 3건"
  • 프로필 아이콘: aria-label="내 프로필"
  • SVG 아이콘: aria-hidden="true" (부모 링크가 라벨을 가지므로)
  • 검색 input: aria-label="매물 검색"

4-C. 하단 탭바 접근성 (모바일)

개선:

  • <nav aria-label="하단 메뉴"> 랜드마크
  • 활성 탭: aria-current="page"
  • 각 탭에 aria-label (예: "마켓 - 현재 페이지")
  • SVG 아이콘: aria-hidden="true"
  • focus-visible 스타일: focus-visible:ring-2 focus-visible:ring-gold
  • 알림 뱃지 (채팅 탭): aria-label에 건수 포함

4-D. 키보드 단축키 (PC 데스크탑)

데스크탑 사용자를 위한 단축키
⌘K / Ctrl+K 검색 포커스
Escape 모달/검색 닫기
? 단축키 도움말 표시

→ useKeyboardShortcuts 커스텀 훅으로 구현
→ input/textarea 포커스 중에는 비활성화
→ 최소한의 단축키만 제공 (과도한 단축키는 오히려 혼란)

4-E. 네비게이션 뱃지 (읽지 않은 알림/채팅)

현재: 네비게이션에 읽지 않은 수 표시 없음

개선:

  • 헤더 알림 아이콘: 빨간 dot 뱃지 (읽지 않은 알림 있을 때)
  • 모바일 채팅 탭: 숫자 뱃지 (읽지 않은 메시지 수)
  • SSE new_message 이벤트로 실시간 업데이트
  • 뱃지 숫자를 aria-label에 포함하여 스크린 리더 접근
